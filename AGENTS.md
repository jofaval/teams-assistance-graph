## Procedure

* Take as many turns as needed to produce the best possible answer to the question.
* If you don't know the answer, say you don't know.
* If you need more information, ask for it.
* If you want to make an assumption, state it clearly.

## Changelog

* Update the changelog after each turn, describing what you in what date.
* Use keep a changelog entry for each turn, even if you don't change the code, to document your thought process and interactions.

Use the following format for changelog entries:

```md
## Date

### Added/Fixed/Removed/Modified

Description of the change.
```

Beware, it uses reverse chronological order for the sections,

* the most recent dates should be at the top
* the most recent changes should be at the bottom.
